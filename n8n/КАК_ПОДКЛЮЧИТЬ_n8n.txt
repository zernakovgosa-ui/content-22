ЧТО ЭТО
--------------------------------
n8n — это конструктор автоматизаций. Мы соберём цепочку:
   по расписанию -> задать тему -> вызвать TREZZY API ->
   API сам пишет сценарий (OpenAI) + делает ролик + субтитры ->
   ролик готов к публикации.

У тебя n8n уже стоит на http://localhost:5678 (мы его раньше не трогали).


ВАЖНО ПРО АДРЕС
--------------------------------
n8n обычно крутится в Docker. Изнутри Docker "127.0.0.1" — это
сам контейнер, а НЕ твой компьютер. Поэтому в узлах n8n адрес
нашего API пишется так:

   http://host.docker.internal:8001/generate-from-plan

(НЕ 127.0.0.1 и НЕ localhost — иначе n8n не достучится до завода).

Если n8n стоит НЕ в Docker, а напрямую — тогда можно
http://127.0.0.1:8001/generate-from-plan


ШАГ 1. УБЕДИСЬ ЧТО ЗАВОД ЗАПУЩЕН
--------------------------------
Дважды кликни START.bat. Дашборд должен открыться на :8001,
плашки API и Worker — зелёные.


ШАГ 2. ИМПОРТИРУЙ ГОТОВУЮ СВЯЗКУ В n8n
--------------------------------
1. Открой http://localhost:5678
2. Сверху справа -> три точки (…) -> Import from File
   (или: Workflows -> Import)
3. Выбери файл:
   C:\trezzy-content-factory\n8n\trezzy_pipeline.json
4. Связка загрузится. Увидишь узлы:
   "Каждые 6 часов" -> "Тема ролика" -> "Создать ролик (TREZZY API)" -> "Успешно?"


ШАГ 3. ПРОВЕРЬ ТЕМУ
--------------------------------
Открой узел "Тема ролика" — там поле topic.
Впиши тему, которую хочешь (например "Партнёрская программа TREZZY").
render_mode = avatar (говорящий герой) или fast (быстрый ролик).


ШАГ 4. ТЕСТ-ПРОГОН (вручную, без расписания)
--------------------------------
1. Нажми на узел "Создать ролик (TREZZY API)"
2. Нажми "Execute step" (или "Test workflow" вверху)
3. Подожди (avatar ~1 мин, fast ~15 сек)
4. Если справа пришёл ответ со "status": "success" — РАБОТАЕТ.
   Ролик появится в C:\trezzy-content-factory\output\latest\final.mp4


ШАГ 5. ВКЛЮЧИ АВТОМАТ
--------------------------------
Когда тест прошёл — переключи workflow в "Active" (тумблер вверху).
Теперь каждые 6 часов завод сам делает ролик.
Интервал меняется в узле "Каждые 6 часов".


ШАГ 6. ПУБЛИКАЦИЯ (узел добавляешь сам)
--------------------------------
После узла "Успешно?" добавь узел публикации:

 - Telegram (БЕЗОПАСНО, рекомендуется):
   узел Telegram -> Send Video. Бот-токен от @BotFather,
   chat_id твоего канала. Это официальный API, банов нет.

 - Instagram / TikTok / YouTube:
   ТОЛЬКО через их официальные API (узлы требуют регистрации
   приложения и токенов). Автопостинг через логин/пароль
   запрещён и ведёт к бану аккаунта — так не делаем.

 - Ручная очередь:
   ничего не добавляешь. Готовые ролики лежат в output\jobs\,
   публикуешь сам когда удобно.


ЕСЛИ n8n НЕ ДОСТУЧАЛСЯ ДО API
--------------------------------
Ошибка connection refused / ECONNREFUSED:
 1. Проверь что адрес = host.docker.internal:8001 (не localhost).
 2. Проверь что START.bat запущен и дашборд открывается на :8001.
 3. Если n8n не в Docker — поменяй адрес на 127.0.0.1:8001.
