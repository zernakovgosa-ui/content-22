## 0. TL;DR для новой сессии
В репозитории ДВЕ независимые системы:
1. **TREZZY Content Factory** (порт 8001) — завод рекламных шортсов для парфюм-бренда. Запуск `START.bat`/`RUN.bat`.
2. **Нарезчик / Clipper** (порт 8002) — отдельный инструмент: YouTube-ссылка/файл → нарезка на шортсы → одобрение в Telegram → распределение по аккаунтам → автопостинг на YouTube → статистика → продажа каналов на залёте. Запуск `CLIPPER.bat`. **Сейчас активная работа идёт здесь.**

Обе используют общий движок (`packages/`), общий `data/settings.json`, общий venv и общий ffmpeg (imageio-ffmpeg).

## 1. КАК ЗАПУСКАТЬ (Windows, путь кириллический!)
- venv: `trezzy-video-worker/.venv/Scripts/python.exe` (системный Python 3.13).
- `START.bat` — завод (8001): hard-kill старого процесса + чистка `__pycache__` + venv + uvicorn. Дашборд `http://127.0.0.1:8001`.
- `RUN.bat` — только поднять завод (venv уже есть).
- `CLIPPER.bat` — нарезчик (8002): hard-kill порта 8002, чистка pycache, uvicorn `clipper.server:app`. Дашборд `http://127.0.0.1:8002`.
- `TEST_CLIP.bat` — быстрый тест нарезки локального файла (перетащить видео).
- `STOP.bat` — остановить.

### 🔴 ЖЕЛЕЗНОЕ ПРАВИЛО
**После ЛЮБОЙ правки Python-кода — перезапуск .bat.** Сервер держит код в памяти; страница в браузере обновляется, а процесс нет. Симптом «нажал — ничего» / «404 на новом эндпоинте» = крутится старый код, перезапусти .bat. (В дашборде нарезчика уже есть авто-подсказка про это.)

## 2. КРИТИЧНЫЕ ГРАБЛИ (новый агент ДОЛЖЕН это знать)
1. **Кириллический путь** (`Desktop\контент завод`) ломает C-сборки. Зависимости ставить ТОЛЬКО колёсами (`requirements.txt` уже wheel-only). numpy/moviepy старых версий не ставятся на 3.13 → вынесены в `requirements-worker.txt` (опционально). См. `[[py313-requirements-risk]]` логику.
2. **OpenCV не открывает кириллические пути** → каскад лица и кадры читаются через копию в ASCII-temp / `cv2.imdecode`. Уже сделано в `face_crop.py`.
3. **User-Agent ОБЯЗАТЕЛЕН** на всех HTTP-клиентах (`llm_client`, `transcribe`, `stock_client`, `did_client`, `youtube`). Без него Cloudflare перед Groq/Google отдаёт `HTTP 403 / 1010`. Симптом был: «нет субтитров + тупой выбор» — это Groq молча падал.
4. **yt-dlp ищет файл `ffmpeg.exe`**, а бинарь imageio называется иначе → `downloader.py` копирует его в `clipper/data/bin/ffmpeg.exe`. Без этого «ffmpeg is not installed» при 1080p.
5. **Сеть владельца рвёт SSL волнами** (`_ssl.c:1015 handshake timed out`, `UNEXPECTED_EOF`). Это НЕ баг кода. Лечение уже встроено: отложенные ретраи автопоста (10/20/30… мин, до 6 серий ≈ 2.5ч), опциональный `https_proxy` в settings.
6. **Groq бесплатный** используется как LLM везде (порядок провайдера: anthropic → openai → groq → none). Бесплатный тариф = rate limits → в `clip_agent` чанк-скан с ретраями и паузами.

## 3. ЧТО УЖЕ РАБОТАЕТ

### Завод (8001), `apps/api/main.py`, `apps/dashboard/index.html`
- `render_mode`: **fast** (текст на фоне), **avatar** (D-ID говорящая голова), **clip** (нарезка), **real** (живые стоковые кадры Pexels). Диспетчер в `_render_one`.
- Агенты-пайплайн (`packages/agents/`): BaseAgent.run() пробует `_llm()`, при любой ошибке падает в `_local()` (шаблон). **НЕЛЬЗЯ ломать этот фолбэк.**
- Защищённые id/классы дашборда (НЕ трогать): `factory, factory-led, factory-state-label, fx-queue, fx-done, fx-mode`, классы анимации завода `.pw .bodyG .legL .legR .armL .armR .walker .clip-travel .belt-dash .reel .pub-pop`.

### Нарезка (clip-движок) — `packages/video/clip_renderer.py`, `packages/agents/clip_agent.py`
- **Выбор моментов v2 (как OpusClip):** Этап A — Groq сканит ВЕСЬ транскрипт чанками, оценивает кандидатов по рубрике (hook 0-30 / flow 0-25 / value 0-20 / emotion 0-15 / quote 0-10) + точная фраза-хук. Этап B (детерминированный) — снап старта на начало предложения с хуком, конца — на конец предложения; **клип 10–60с, заканчивается ТОЛЬКО на завершённой мысли** (нет конца в окне → кандидат отброшен, без обрывов); дедуп по баллу; топ-N в порядке рейтинга (clip_01 = лучший).
- **Рендер v3 (single-pass, быстрый):** исходник → crop/hqdn3d/lanczos/cas → субтитры через **ffconcat «слайд-ленту»** (один overlay вместо сотен) → один финальный энкод. Промежуточных гигабайтных файлов НЕТ. Клипы рендерятся парами (ThreadPool(2)). Замер: 2×35с 1080p 313с → **166с**.
- Субтитры: **караоке** (активное слово золотом), крупные с обводкой, наложений нет по построению. Хук-заголовок сверху ОТКЛЮЧЁН (владелец просил убрать).
- Кадр: **fill** (на весь экран, без рамок) по умолчанию; есть `wide` (размытые поля) — владельцу не зашёл.
- `render_quality`: **max** (дефолт, crf15+tune film) / **fast**. Есть самопроверка декодом + таймаут `_run`.

### Реалистичный режим (real) — `packages/video/stock_renderer.py`, `stock_client.py`, `agents/stock_director.py`
- Pexels Video API (нужен бесплатный `pexels_api_key` — у владельца ЗАДАН). Качает живые вертикальные кадры по англо-запросам (StockDirector через Groq) → нормализует 9:16 → склейка → субтитры → опц. музыка из `assets/music/`.

### Нарезчик (8002) — `clipper/`
- `server.py` (FastAPI, 4 daemon-потока: worker нарезки, TG-поллер, scheduler 60с, stats 4ч), `planner.py` (чистая логика), `dashboard.html` (кремовый стиль, 4 вкладки), `downloader.py` (yt-dlp), `publishers/youtube.py` (Data API v3).
- Полный цикл: **YouTube-ссылка** (или файл в `clipper/sources/{видосы,фильмы,сериалы}`) → скачать лучшее ≤1080p + прочитать описание ролика (контекст для названий шортсов) → транскрипция → ClipAgent → render_clips → **каждый клип в Telegram с ✅/❌** → одобренные распределяются равномерно по аккаунтам категории (2/день, 11:00+18:00, лучшие первыми) → в слот **автозалив на YouTube** → ✅ ссылка в TG → статистика тянется сама каждые 4ч → **🚀 на 80k**, **💰 «канал готов к продаже» на 400k**.
- Аккаунт с платформой `youtube` + client_id/secret → OAuth «🔗 Подключить» (loopback `http://localhost:8002/auth/yt/callback`). Прогрев МЯГКИЙ: новый аккаунт сразу в работе (`warmed=True`), 🧊 — ручной тормоз.
- **Бизнес-модель владельца:** аккаунты-расходник, растим до ~400k просмотров и продаём канал на маркетплейсе.

## 4. ❗ ЧТО ЕЩЁ ПРЕДСТОИТ СДЕЛАТЬ (главное)

### П0. Метод скачивания (РЕШЕНО → self-host cobalt) — срез 2026-06-17
Живой тест резолверов показал: **invidious и piped МЕРТВЫ как качалки** (домены
отвечают 200, но видео-потоков не отдают — YouTube прижал). **cobalt РАБОТАЕТ**
(`fox.kittycat.boo`, `rue-cobalt.xenon.zone` выдают туннель), но публичные инстансы
ротируются и 2/5 уже просят логин. yt-dlp напрямую душится РФ-DPI без VPN.
- Цепочка в `data/settings.json` урезана: `yt_download_chain = ["cobalt", "ytdlp"]`
  (мёртвые invidious/piped убраны, чтобы не висеть в пустых таймаутах). cobalt-инстансы
  переставлены: рабочие первыми.
- **План владельца: поднять СВОЙ cobalt** (он выбрал этот путь). Готовые файлы —
  `deploy/cobalt/` (docker-compose с cobalt v11 + yt-session-generator poToken-сайдкар +
  watchtower, keys.json, README с пошаговой установкой на VPS вне РФ). После деплоя:
  прописать `cobalt_self_host` + `cobalt_api_key` в settings.json. Поля в коде уже есть.
- ВАЖНО про YouTube-бан датацентровых IP: лечится сайдкаром `yt-session-generator`
  (poToken генерится на IP VPS). Без него self-host ловит `error.api.youtube.login`.
