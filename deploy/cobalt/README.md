# Своя cobalt-качалка (self-host) для клиппера

Цель: личная качалка YouTube, которая **не зависит от чужих дохнущих инстансов** и
**обходит РФ-DPI**. Клиппер коннектится к твоему VPS (обычный HTTPS/HTTP), а уже VPS
тянет видео с ютуба и отдаёт тебе байты. googlevideo при этом РФ-DPI не видит.

> Почему именно так: публичные cobalt-инстансы (fox/rue и т.п.) сегодня живы, завтра
> легли или начали требовать логин. Свой инстанс с poToken-сайдкаром качает стабильно.

---

## Что нужно
- **VPS ВНЕ РФ** (это обязательно — он должен свободно ходить на ютуб).
  Ubuntu 22.04/24.04, 1–2 ГБ RAM, 1 vCPU. ~3–5 €/мес (Hetzner, Contabo, Aeza, 4vps…).
  Бери локацию с хорошим пингом из РФ (Германия/Финляндия/Нидерланды/Турция).
- 10 минут времени. Девопс-опыт не нужен — всё по шагам.

> ⚠️ Датацентровые IP ютуб иногда уже банит. Сайдкар `yt-session-generator` это лечит
> (генерит poToken на IP VPS). Если конкретный IP забанен жёстко — пересоздай VPS /
> возьми другую локацию.

---

## Шаги

### 1. Заведи VPS и зайди по SSH
Запиши **публичный IP** (далее `IP_VPS`). Подключись:
```sh
ssh root@IP_VPS
```

### 2. Поставь Docker (одна команда)
```sh
curl -fsSL https://get.docker.com | sh
```

### 3. Залей сюда эти два файла
Создай папку и положи в неё `docker-compose.yml` и `keys.json` из этой папки проекта
(`deploy/cobalt/`). Скопировать можно через `scp` с ноута или `nano` прямо на VPS:
```sh
mkdir -p ~/cobalt && cd ~/cobalt
# затем создай docker-compose.yml и keys.json (nano) и вставь содержимое
```

### 4. Сгенерь свой ключ-UUID и впиши в keys.json
На VPS:
```sh
cat /proc/sys/kernel/random/uuid
```
Скопируй вывод (напр. `3f9a...`), замени им `PASTE-YOUR-OWN-UUID-HERE` в `keys.json`.
**Этот UUID = твой `cobalt_api_key`** (понадобится в settings.json, шаг 8). Не теряй.

### 5. Впиши свой адрес в docker-compose.yml
Замени `http://CHANGE_ME_VPS_IP:9000/` на `http://IP_VPS:9000/`
(в обоих случаях — со слешем на конце).

### 6. Открой порт 9000 в фаерволе
```sh
ufw allow 22/tcp && ufw allow 9000/tcp && ufw --force enable
```
Если у провайдера есть свой фаервол в панели — там тоже открой TCP 9000.

### 7. Запусти
```sh
docker compose up -d
docker compose logs -f cobalt      # Ctrl+C чтобы выйти из логов
```
Watchtower дальше сам обновляет образы.

### 8. Проверь с ноута, что качает
Подставь свой IP и UUID:
```sh
curl -X POST http://IP_VPS:9000/ ^
  -H "Content-Type: application/json" ^
  -H "Authorization: Api-Key ТВОЙ-UUID" ^
  -d "{\"url\":\"https://youtu.be/jNQXAC9IVRw\"}"
```
Ждём ответ вида `{"status":"tunnel","url":"http://IP_VPS:9000/tunnel?..."}`.
Если так — качалка работает. 🎉

### 9. Подключи к клипперу
В `data/settings.json` проекта пропиши (значения — свои):
```json
"cobalt_self_host": "http://IP_VPS:9000",
"cobalt_api_key": "ТВОЙ-UUID",
```
`yt_download_chain` уже стоит `["cobalt", "ytdlp"]` — твой self-host берётся ПЕРВЫМ
и доверенным (без пробы). Перезапусти `CLIPPER.bat`.

## Если что-то не так
- **`error.api.youtube.login` в ответе** → сайдкар не поднялся или IP забанен.
  Проверь `docker compose logs yt-session-generator`. Первый токен генерится не сразу
  (запускается headless-хром) — подожди минуту и повтори. Не помогло — другой VPS/локация.
- **Висит/таймаут с ноута** → порт 9000 закрыт в фаерволе (шаг 6) или провайдерская панель.
- **Качает, но медленно** → видео идёт ютуб→VPS→ноут (двойной транзит). Возьми VPS с
  каналом пожирнее / поближе.

## Безопаснее (опционально, потом)
- Сейчас ключ ходит по HTTP в открытую. Для долгой эксплуатации повесь домен + HTTPS
  (reverse-proxy Caddy авто-TLS), тогда `API_URL` и `cobalt_self_host` → `https://...`.
- Можно прибить ключ к своему IP: в `keys.json` у UUID добавь `"ips": ["твой_дом_IP"]`
  (но домашний IP обычно динамический — тогда не надо).
